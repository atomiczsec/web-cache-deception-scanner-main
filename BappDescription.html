Web Cache Deception Scanner is a Burp Suite extension that automatically detects Web Cache Deception (WCD) vulnerabilities in web applications.

This extension adds a dedicated Active Scanner check and a context menu option for targeted manual testing.

About Web Cache Deception

Web Cache Deception is a critical vulnerability first described by security researcher Omer Gil in 2017. It exploits misconfigurations in web caching systems, allowing attackers to trick caches into storing and serving sensitive, user-specific content as if it were public static resources. This can lead to the exposure of private data, authentication tokens, or other confidential information to unauthorized users.
Read Omer Gil's original write-up

  • The attacker crafts a URL that appends a fake static file extension (e.g., /profile/secret.jpg) to a sensitive endpoint.
  • The victim, while authenticated, visits this URL, causing the cache to store the sensitive content under the deceptive path.
  • The attacker then accesses the same URL and receives the cached sensitive data.

Impact: Successful exploitation can result in theft of personal information, authentication tokens, CSRF tokens, account data, or even enable further attacks such as Cross-Site Request Forgery or account takeover.
Learn more about WCD impacts

Mitigation: Ensure cache servers respect application Cache-Control headers, and avoid caching decisions based solely on URL patterns or file extensions. Sensitive resources should never be cached as public content.
